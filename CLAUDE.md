Orientation for coding agents working in this repo. The human-facing README.md

## What this project is now

A spatially explicit assessment of Pacific yew (*Taxus brevifolia*) habitat decline
across British Columbia, built from Google AlphaEarth satellite embeddings classified
by XGBoost. Two main deliverables:

- **The paper** ‚Äî `docs/PAPER_FINAL_DRAFT.md` (source of truth) ‚Üí `docs/PAPER_FINAL_DRAFT.pdf`.
- **The web map** ‚Äî `docs/index.html` + `docs/tiles/` (GitHub Pages, Leaflet).

`docs/METHODS_AND_RESULTS_DRAFT.md` is an **older** draft; edits generally go to
`PAPER_FINAL_DRAFT.md` only. Watch for drift between them.

## Geospatial data sources (BC Data Catalogue WFS)

Province-wide layers are fetched live from `https://openmaps.gov.bc.ca/geo/pub/wfs`
(full resolution, EPSG:4326, `count=10000`). Key typeNames:

- `WHSE_TANTALIS.TA_PARK_ECORES_PA_SVW` ‚Äî provincial parks / ecological reserves /
  protected areas / recreation areas (~930 features)
- `WHSE_TANTALIS.TA_CONSERVANCY_AREAS_SVW` ‚Äî conservancies (~169)
- `WHSE_ADMIN_BOUNDARIES.CLAB_NATIONAL_PARKS` ‚Äî national parks in BC (~102 polygons)

`scripts/analysis/build_park_contours.py` combines these into
`docs/tiles/park_contours.geojson` (the web-map layer). Overpass/OSM is **unreachable**
in this environment ‚Äî use the BC WFS national-parks layer instead.

## The tile cache gotcha (read before any per-tile analysis)

There are **99 study tiles** (85 coastal + 14 ICH), but:

- Raw `.npy` grids + embeddings exist for only **42** tiles in
  `results/analysis/cwh_spot_comparisons/tile_cache/` (`{slug}_grid.npy`,
  `{slug}_logging.npy`, `{slug}_emb.npy`). The other 57 were cleaned up after the
  web PNGs were rendered, and the embeddings are NOT cached ‚Äî they can't be
  reclassified without re-pulling from GEE.
- **All 99** tiles have suppressed-probability PNGs + per-tile stats:
  `docs/tiles/{slug}.png` (+ `_logging.png`) and `docs/tiles/tiles.json`
  (`bbox`, `stats.p50_ha`, logging breakdown).

To analyse all 99 tiles, recover the habitat mask from the PNGs:
`scripts/analysis/protected_area_from_tiles.py` inverts the export colormap
(`YEWCMAP`) back to suppressed probability (validated to ~1% of stored `p50_ha`),
rasterises the parks layer per tile, and writes
`results/analysis/protected_area_all_tiles.json`. This is the reproducible,
no-GEE path for tile-level stats.

`scripts/analysis/yew_in_protected_areas.py` is the older analysis ‚Äî it only sees
the 42 cached grids, so its totals are a biased subset. Prefer the all-tiles script.

## Conventions & gotchas

- Pixel area convention is **0.01 ha/px** (treats every pixel as 10 m) regardless of
  the PNG's actual geographic pixel size ‚Äî protected *fractions* are pixel ratios so
  this cancels, but absolute ha use 0.01.
- "Probability mass" = Œ£(per-pixel probability) √ó 0.01 ha ‚Äî not a pixel count.
- Three distinct habitat figures exist on purpose: 47,534 ha (continuous mass, the
  headline), 37,885 ha (P‚â•0.5 threshold, protected-area denominator), 29,028 ha
  (water-grid 42-tile subset). Don't conflate them.
- Suppression categories (VRI logging): 1 water, 2 logged<20, 3 logged 20‚Äì40,
  4 logged 40‚Äì80, 5 forest 80‚Äì150, 6 alpine, 7 old-growth(>150). Only cat 7 is yew
  habitat for loss estimation.
- Don't reintroduce the "267 manual annotations weighted 3√ó as positives" claim ‚Äî
  that file (`data/raw/yew_annotations_combined.csv`) is 267 rows but only 64 are
  positive; the production positives are 1,043 iNaturalist records.
- `data/.~lock.*` files are stale LibreOffice locks ‚Äî safe to ignore/remove.
